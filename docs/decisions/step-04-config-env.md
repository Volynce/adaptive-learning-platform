\# Step-04: Конфигурация через env (Settings + .env.example)



\## Решения

1\) Конфигурация приложения задаётся через переменные окружения (env-first).

2\) Для локальной разработки используется .env (не коммитится), шаблон .env.example коммитится.

3\) Подключение к БД задаётся только через DATABASE\_URL (единый формат).

4\) Настройки читаются строго через типизированный Settings (pydantic-settings).

5\) Добавлен api/deps.py как единая точка FastAPI dependencies (пока только Settings).



\## Зачем

\- Исключить расхождения конфигурации между окружениями.

\- Упростить воспроизводимость при откатах git и при запуске через Docker Compose.

\- Обеспечить “одну точку правды” для DATABASE\_URL и JWT параметров.



\## Критерий готовности

\- В репозитории есть .env.example.

\- Есть Settings и get\_settings() (pydantic-settings).

\- Импорт Settings проходит; инстанцирование работает при заданных env-переменных.
